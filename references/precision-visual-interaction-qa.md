# 精準視覺、互動動畫、資料查核與品質驗證

實際建立或修改任何數位互動教材時讀取本文件。它是跨學科執行規範，不限特定物種、自然科或七頁籤架構。

## 1. 學習契約與證據矩陣

在繪圖或寫互動前，建立以下資料：

- 教材主題、年級、先備知識與使用情境。
- 2–5 個核心學習目標。
- 學生完成後應能辨認、解釋、比較、推論、計算或操作的表現。
- 每個活動處理的概念與常見迷思。
- 學生真正需要觀察的視覺焦點。
- 不適合動畫化、可能造成認知負荷的內容。

建立證據矩陣：

| 學習目標 | 畫面／互動 | 學生行動 | 正確性依據 | 成功證據 |
|---|---|---|---|---|
| 要學會什麼 | 用何種視覺呈現 | 學生做什麼 | 來源或規則 | 如何判定理解 |

每張圖、每段動畫、每個互動至少對應一項學習目標。無法填入矩陣的裝飾不要加入。

## 2. 資料查核與授權

對以下內容進行查核：

- 人物、動植物、物件、服飾、器物與建築外形。
- 科學構造、裝置、現象、方向、因果與安全操作。
- 公式、數值、比例、幾何、座標、統計與單位。
- 地圖、疆界、地形、位置、路線、方位與距離。
- 歷史年代、事件順序與文化符號。
- 字形、筆順、讀音、音標、文法與例句。
- 法規、標準、資料年度及其他會隨時間改變的資訊。

執行規則：

1. 先用政府、教育機關、博物館、大學、研究機構、官方文件及專業資料庫。
2. 重要概念至少交叉核對兩個可信來源；高風險或有爭議內容增加來源。
3. 對會改變的資訊記錄查核日期與資料年度。
4. 分開標示已確認事實、合理推論、教學示意與比例簡化。
5. 發現原內容疑似錯字、誤名、錯誤數值或不可能因果時，先查核再修正，不照錯誤內容製圖。
6. 外部圖片只作研究與比對。除非授權允許且教材需要，不直接重製或嵌入。
7. 在 `docs/references.md` 記錄名稱、網址、查核日期、使用目的、授權或使用方式。

## 3. 視覺形式選擇

依學習關係選擇最小而有效的視覺：

- **教學寫實圖**：辨認真實外觀、材料、構造、服飾、器物、地貌或生物特徵。
- **示意圖**：呈現力、熱、光、聲、電、氣流、粒子、語法或其他不可直接觀察關係。
- **剖面圖**：呈現內外層、器官、機械、地層或材料結構。
- **流程圖**：呈現三個以上步驟、因果、判斷或狀態轉換。
- **時間軸**：呈現事件先後、週期或歷史發展。
- **地圖**：呈現位置、方向、範圍、路線、距離或空間分布。
- **圖表**：呈現數量、比例、趨勢、分類或分布。
- **動畫**：只有靜態前後圖無法清楚呈現變化、運動、步驟或因果時使用。
- **互動模擬**：學生需要改變條件並觀察可重複、可解釋的結果時使用。

不要把所有內容都畫成圓形、橢圓形、表情符號或相同卡通角色。不要因為能做動畫就增加動畫。

## 4. 通用視覺品質

逐張檢查：

1. 主體完整，未被容器、遮罩或 `viewBox` 裁切。
2. 比例、數量、角度、位置、方向與順序符合題意。
3. 細節足以支援觀察與比較，同時符合學生年齡。
4. 同一概念在不同頁面維持一致顏色、符號、名稱與尺度邏輯。
5. 背景、陰影、漸層和裝飾不搶走焦點，也不製造錯誤空間判讀。
6. 縮小到手機後仍能辨認主要結構。
7. 不依賴圖片內極小文字才能理解。
8. 寫實需求不以通用圖示代替；示意需求不假裝成真實比例。
9. 生活情境使用新的應用場景，不只重複核心大圖。
10. 同一視覺中的線寬、標籤框、箭頭端點、色彩和層次一致。

## 5. 標示與文字安全區

1. 優先把標籤放在主體輪廓外，以短引線指出正確位置。
2. 引線不要穿越其他重要構造、資料點、地名、座標軸或控制元件。
3. 文字不要遮住主體、臉部、辨識特徵、圖例、刻度或動畫路徑。
4. 箭頭方向必須符合作用、移動、閱讀或時間方向。
5. 標籤與背景保持足夠對比；同類標籤維持一致樣式。
6. 空間不足時重構版面、分圖或改用外部 HTML 說明，不只縮小字體。
7. 手機版重新安排文字與圖像，不只等比例縮小桌機構圖。
8. 動畫物件若會移動，引線應位於相同座標系統，或把振幅限制在仍能正確指向的範圍。

## 6. 動畫教學設計

製作前回答：

- 學生要看見哪個變化？
- 哪一部分改變？哪些條件保持不變？
- 變化的原因、順序與結果是什麼？
- 動畫結束後學生要提出哪項證據或推論？

執行規則：

1. 一次聚焦一個主要概念；多步驟內容分階段播放。
2. 同步更新階段文字，保留可比較的前後狀態。
3. 不為效果改變原本應固定的條件。
4. 方向、距離、角度、速度、比例及時間若具教學意義，必須正確。
5. 速度足以觀察，避免快速閃爍、無意義循環及同時搶焦。
6. 暫停時仍是一張完整、可理解的教學圖。
7. 提供播放、暫停、重播；複雜內容提供逐步或速度調整。
8. 動畫主體不離開可視範圍、不撞到文字、不遮住操作控制。
9. 動畫後加入觀察問題、證據整理或推論任務。
10. `prefers-reduced-motion` 下保留完整靜態資訊與可操作性。

## 7. SVG/CSS/JavaScript 技術規範

### SVG

- 設定適當 `viewBox`，避免依賴固定像素尺寸。
- 同頁多張 SVG 的 gradient、mask、filter、pattern、clipPath ID 必須唯一。
- 只需要線條的開放路徑明確設定 `fill="none"`，避免自動閉合產生黑色區塊。
- 控制描邊寬度，避免縮小後消失或變得過粗。
- 重要 SVG 使用 `role="img"`，加入精確 `aria-label` 或 `<title>`。

### Transform

CSS animation 不要覆寫 SVG 原本的 transform 定位。使用雙層群組：

```html
<g class="animated-layer">
  <g transform="translate(...) scale(...)" aria-hidden="true">
    <!-- 視覺主體 -->
  </g>
</g>
```

- 外層負責動畫，內層保留基礎定位。
- 明確設定 `transform-origin`。
- 先計算最大位移、旋轉邊界與縮放後包圍盒，避免超出 `viewBox`。
- 不用動畫掩蓋原本的定位錯誤。

### 狀態控制

- 播放、暫停、重播、速度、分步控制必須連接真實狀態。
- 切換頁籤、題目或場景時停止不再可見的計時器與動畫。
- 重播回到可預期初始狀態。
- 畫面、文字、數值與無障礙狀態同步更新。
- 隨機題目及模擬加入邊界條件、固定測試案例或可重複種子。

## 8. 跨學科精準度

### 數學與資料

- 幾何比例、角度、位置與題意一致；正確標示等長、平行、垂直等關係。
- 座標點、圖形變換、數線刻度、等分區塊與百分比面積精確。
- 圖表資料與原始數值一致；座標軸、基準線與截斷不造成誤導。
- 動畫中的數值、圖形、單位和文字同步更新。
- 檢查零、負數、最大最小值、重複值及無解／多解狀況。

### 自然科學

- 力、光、熱、電、聲、運動、粒子及物質變化符合相關定律與已知現象。
- 非真實比例明確標示「示意」；不混淆微觀與巨觀層次。
- 操作變因、控制變因、應變變因清楚，結果可重複。
- 不為動畫效果製造錯誤因果；簡化、限制與不確定性明確說明。
- 高風險科學活動加入安全限制及極端條件測試。

### 地圖、歷史與文化

- 方位、位置、比例、範圍、路線與時代疆界正確；記錄資料日期。
- 歷史地圖、事件順序、服飾、器物、建築與交通工具符合指定時代。
- 不混用不同文化或時代符號，不用刻板印象代替查核。
- 有爭議內容區分事實、來源、解釋與觀點。
- 時間軸刻度合理，地圖標籤不遮住重要位置。

### 語文與符號

- 查核字形、標點、筆順、讀音、注音、拼音、音標、文法與例句。
- 筆順動畫依正式規範逐筆播放，路徑和書寫方向一致。
- 文法標示對準正確詞語或成分；句子重組保留合理語意。
- 插圖協助理解情境，不提前洩漏需要閱讀推論的答案。
- 多語內容檢查字型、斷行、標點與閱讀方向。
- 配對題若有多個合理答案，不可只接受未說明的單一答案。

### 藝術、健康與生活

- 查核作品、樂器、色彩、文化背景與技法，不用相似圖案冒充原作。
- 示範動作保留正確姿勢、順序、工具及安全距離。
- 健康、安全、飲食與生活技能依可信官方指引，避免用視覺暗示絕對效果。
- 情緒與人際情境避免污名、單一刻板反應及不必要的性別／文化偏見。

## 9. 互動與生活情境

每個互動要求學生進行至少一項有意義行動：觀察、分類、排序、配對、比較、預測、操作變因、蒐集證據、建立模型、解釋或修正。

互動必須：

1. 清楚說明任務與操作方式。
2. 提供即時但不直接洩漏答案的回饋。
3. 錯誤回饋指出重新觀察或思考方向。
4. 可以重設並重新操作，不因誤觸丟失重要進度。
5. 不要求過度精細的拖曳；觸控區域足夠大。
6. 完成後提供概念整理或遷移問題。

生活應用圖使用學生可觀察的新場景，保留正確結構、數據或因果，不以複雜背景分散注意，也不直接顯示完整答案。

## 10. 無障礙

- 重要圖像的替代文字描述主體、動作、關係與教學目的，不只寫「教材圖片」。
- 不只靠紅綠色、透明度或動畫區分狀態。
- 按鈕、頁籤、滑桿、拖曳替代操作及播放控制支援鍵盤。
- 使用清楚焦點、語意標籤、狀態文字與適當 `aria-live`。
- 文字和背景維持可讀對比，放大或手機斷行後不遺失內容。
- 降低動畫模式停用非必要動作，但保留資訊、步驟與控制。

## 11. 瀏覽器驗證與證據

最低 viewport：

- 桌機：1440×1000
- 平板：768×1024
- 手機：390×844

對每個主要頁面和活動檢查：

1. 有實質內容，沒有框架錯誤覆蓋層。
2. 沒有未處理主控台錯誤或重要失敗請求。
3. 沒有水平溢出、裁切或控制重疊。
4. 文字不壓圖，標籤和引線指向正確。
5. 動畫初始、中間、結束和重播狀態正常。
6. 播放、暫停、速度、頁籤切換與重設有效。
7. SVG 沒有意外填色、重複 ID 衝突或 transform 覆寫。
8. 圖表、座標、資料、單位和圖例一致。
9. 手機文字可讀，底部導覽不遮住後排內容。
10. 鍵盤、觸控及降低動畫模式可完成主要流程。

保存證據：

- 每個主要活動至少一張桌機截圖。
- 重要響應式或高風險活動至少一張手機截圖。
- 動畫至少抽查初始與一個變化後狀態。
- 記錄 build、audit、placeholder、console、network、overflow 與互動結果。
- 在 `docs/test-report.md` 記錄環境、尺寸、通過項目、修正事項及剩餘限制。

## 12. 失敗修正與部署閘門

發現問題時：

1. 保存錯誤畫面、主控台或失敗請求證據。
2. 分類為內容、資料、構圖、SVG、CSS、互動狀態、響應式、無障礙或部署問題。
3. 修正根因，不用隱藏、裁掉或加裝飾遮掩。
4. 重新正式建置。
5. 以相同 viewport、操作步驟和資料重測。

以下情況不得部署：

- 核心內容、時效資料或安全資訊尚未查核。
- 有已知事實、公式、比例、方向、順序、因果或答案錯誤。
- 使用授權不明資產。
- 文字遮住重要資訊，或任一必要 viewport 有裁切、溢出。
- 動畫跑出畫面、狀態不同步、控制無效或出現意外 SVG 填色。
- 鍵盤或觸控無法完成主要流程。
- 正式 build、資料載入、主控台或關鍵請求失敗。
- 版本、文件、repo 名稱、社群 metadata 或公開網址未同步。
- 未實際檢查公開部署。

## 13. 完成定義

只有全部成立才宣告完成：

- 學習目標、活動與成功證據明確對應。
- 重要內容有可信來源，事實、推論與示意分開。
- 視覺完整、詳細、精準、適齡且沒有無關裝飾。
- 動畫符合概念、因果、空間與時間關係。
- 互動要求有意義的學生思考並有可用回饋。
- 標籤、文字、圖像和控制沒有重疊。
- 無障礙、鍵盤、觸控及降低動畫模式可用。
- 桌機、平板、手機實際瀏覽器驗證通過。
- 正式建置、安全檢查、資料載入與版本文件通過。
- 使用資料夾名稱或英文教材名稱的專用 repo。
- 公開網站和重要資源已驗證成功。

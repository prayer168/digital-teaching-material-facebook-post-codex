# Facebook 中英貼文與連結預覽流程

## 1. 交付內容

產生一份可直接複製的 Facebook 貼文，預設包含：

1. 繁體中文版。
2. 英文版。
3. 兩種語言相同的連結、行動呼籲與重要資訊。
4. 若連結屬於使用者控制或目前專案，建立或更新並驗證 Facebook 連結預覽縮圖。

採溫暖、清楚、自然且適合分享的語氣。除非使用者要求宣傳文案，避免過度行銷。

## 2. 輸出格式

除非使用者要求替代版本或說明，最終只交付完成的貼文。使用以下結構：

```text
【繁體中文】
[繁體中文貼文]

【English】
[English post]
```

有專案網址時，兩種語言都要放入相同網址，並將網址置於各語言段落接近末尾處。使用者要求「一鍵複製」時，把完整中英貼文放在同一個 fenced `text` 區塊。

## 3. 寫作規則

- 保留使用者的訊息、受眾與情緒語氣。
- 使用繁體中文，不使用簡體中文。
- 自然翻譯意思，不逐句機械對照。
- 優先使用 2–4 個短段落，避免密集長文。
- 需要時加入溫和行動呼籲，例如「歡迎點進去看看」或 “Take a look here”。
- 除非使用者要求或貼文明顯受益，否則不加 hashtag；需要時限 1–3 個廣泛標籤。
- 除非使用者已使用 emoji 或要求活潑風格，否則不加 emoji。
- 使用者只提供連結或簡短主題時，從可取得的專案內容推導具體而精緻的貼文。
- 教材貼文要具體描述學生能觀察、操作、比較、推論或練習的內容及學習價值，不只寫「歡迎使用」。

長度與風格：

- 「短一點」：每種語言 1–2 個短段落。
- 「正式」：用詞較完整、克制，減少口語。
- 「活潑」：較溫暖、對話感強；適合時可少量使用 emoji。

## 4. Facebook 連結預覽

當使用者要求縮圖、控制該網站，或連結指向目前 workspace／專案時：

1. 建立橫式 `1200 × 630` 圖片，確保中央安全區文字清楚可讀。
2. 優先使用專案自有 SVG、CSS、圖形與文字，不使用授權不明的第三方圖片或影片影格。
3. 加入絕對 HTTPS Open Graph 值：`og:type`、`og:url`、`og:title`、`og:description`、`og:image`、`og:image:width`、`og:image:height`、`og:image:alt`。
4. 加入 `twitter:card=summary_large_image` 與相符的 title、description、image。
5. 驗證公開頁面與圖片均為 HTTP 200，圖片 MIME 正確、尺寸精確，已渲染頁面包含預期 metadata。
6. 目視確認縮圖標題、主體、對比及安全區在縮小預覽時仍清楚。
7. 若已分享網址仍顯示舊圖，提醒使用者到 Meta Sharing Debugger 執行 **Scrape Again**。

若使用者只要求貼文但已有自有／目前專案網址，縮圖建立或更新、metadata 接線、在授權範圍內儲存或部署，以及公開預覽輸入檢查，都屬於預設完成路徑。

不要修改第三方網站。若使用者只要求文案且無權控制連結網站，直接完成貼文；只有確實相關時才簡短指出無法修改或缺少預覽 metadata。

## 5. 教材發布整合

教材已通過部署閘門時：

1. 使用正式公開教材 URL 與縮圖 URL。
2. 確保貼文、`og:title`、`og:description` 和教材實際內容一致。
3. 兩種語言都包含公開網址與自然的行動呼籲。
4. 若是「收工」流程，將最終貼文保存為專案根目錄的 `facebook-post.txt`，納入既定版本與部署流程。
5. 公開頁面、縮圖、metadata 或關鍵互動尚未驗證時，不宣稱發布完成。
